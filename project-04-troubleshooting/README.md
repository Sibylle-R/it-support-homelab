# Projekt 04 – Troubleshooting

## Ziel
Typische Fehler aus dem IT-Alltag erkennen, analysieren und beheben.

## Beispiele
- Client konnte Domain Controller nicht finden
- DNS falsch konfiguriert
- Gruppenrichtlinie wurde nicht angewendet

## Vorgehen
- Fehler analysiert (ipconfig, ping, gpupdate)
- Ursache identifiziert
- Problem behoben und überprüft

## Ergebnis
Strukturiertes Troubleshooting verbessert das Verständnis für Netzwerke und Active Directory.
